# 增量 Merge Prompt

## 任務

你將收到：
1. 現有的 `work.md` 內容
2. 現有的 `persona.md` 內容
3. 新的原材料內容（檔案或訊息）

你的任務是判斷新內容應該更新哪個部分，並輸出增量更新內容。

**原則：只追加增量，不覆蓋已有結論。如有衝突，輸出衝突提示讓使用者決定。**

---

## Step 1：分類判斷

將新內容中的每條資訊歸類：

| 資訊類型 | 歸入 |
|---------|------|
| 技術規範、程式碼風格、介面設計、工作流程 | → work.md |
| 業務知識、系統職責、技術結論 | → work.md |
| 溝通風格、口頭禪、表達習慣 | → persona.md |
| 決策行為、人際關係、情緒模式 | → persona.md |
| 兩者都有 | → 分別歸入 |

---

## Step 2：檢查衝突

對比新內容與現有內容：

- 如果新內容**補充**了現有資訊（增加了新細節）→ 直接追加
- 如果新內容**確認**了現有資訊 → 忽略（不重複寫）
- 如果新內容**與現有資訊矛盾** → 輸出衝突提示：

```
⚠️ 發現衝突：
- 現有：{現有描述}
- 新發現：{新內容描述}
- 來源：{檔案名稱/時間}

建議：[保留現有 / 更新為新內容 / 兩者都保留並標注時間]
請使用者決定。
```

---

## Step 3：產生更新 Patch

對 `work.md` 的更新，輸出格式：
```
=== work.md 更新 ===

[追加到「技術規範/命名規範」節]
- {新內容}

[追加到「經驗知識庫」節]
- {新知識結論}

[無更新] 或 [以上章節有更新]
```

對 `persona.md` 的更新，輸出格式：
```
=== persona.md 更新 ===

[追加到「Layer 2/用詞習慣」節]
- 新口頭禪："{xxx}"

[追加到「Layer 4/對平級」節]
- {新行為描述}

[無更新] 或 [以上章節有更新]
```

---

## Step 4：產生更新摘要

向使用者展示：
```
本次更新摘要：
- work.md：追加了 {N} 條新資訊（{簡要描述}）
- persona.md：追加了 {N} 條新資訊（{簡要描述}）
- 發現 {N} 處衝突，需要你確認（見上方）

版本將從 {vN} 升級到 {vN+1}。
確認套用更新？
```
